# Інструкція адміністратора

## 1. Призначення ролі адміністратора
Адміністратор налаштовує користувачів, ролі, довідники, резервне копіювання та контролює якість даних.

## 2. Як створити першого адміністратора

```bash
python manage.py createsuperuser
```

## 3. Як ініціалізувати ролі

```bash
python manage.py init_roles
```

Команда створює групи **Адміністратор складу**, **Комірник**, **Перегляд / аудитор** і призначає базові дозволи.

## 4. Як керувати довідниками
Використовуйте **Керування → Довідники**. Архівуйте непотрібні записи замість фізичного видалення. Архівні записи не пропонуються у формах для нових зв'язків.

## 5. Як знайти дублікати

```bash
python manage.py find_duplicates
```

## 6. Резервне копіювання
Перед масовими змінами або оновленнями створюйте резервну копію бази даних і перевіряйте відновлення на тестовому середовищі.

## Штрихкоди, етикетки та принтери

### Автоматична генерація штрихкодів

Штрихкоди створюються сервісом `core/services/barcodes.py`. Для кожного префікса використовується окрема `BarcodeSequence`, блокування `select_for_update()` і транзакція. Формати:

- `ITM0000000001` — номенклатура;
- `WH0000000001` — склад;
- `LOC0000000001` — локація;
- `RCK0000000001` — стелаж.

Якщо запис уже має штрихкод, новий не створюється.

### Налаштування принтерів

1. Перевірте CUPS-принтери на сервері:

```bash
lpstat -p -d
```

2. Надрукуйте тест:

```bash
echo test | lp -d PRINTER_NAME
```

3. У вебінтерфейсі відкрийте **Принтери** (`/settings/printers/`) і додайте запис. Поле **Системна назва** має збігатися з назвою CUPS-принтера.

### Шаблони етикеток

Шаблони налаштовуються на `/settings/label-templates/`. Типовий розмір — 58×40 мм. Можна керувати показом назви товару, внутрішнього коду та тексту штрихкоду.

### PDF-етикетки та кириличні шрифти

Для коректного відображення українських і російських назв товарів у PDF-етикетках на Ubuntu встановіть Unicode-шрифти DejaVu:

```bash
sudo apt install fonts-dejavu-core
```

Після оновлення застосунку виконайте стандартні команди підготовки середовища та перекладів:

```bash
pip install -r requirements.txt
python manage.py compilemessages
```

### Друк і PrintJob

При друці створюється `PrintJob`. Успішний друк переводить запис у `printed`, помилка CUPS — у `failed` із текстом помилки. Сторінка друку не повинна завершуватися 500-помилкою при збоях `lp`.

## Політика паролів

За замовчуванням прості паролі дозволені. Це зручно для ізольованої локальної мережі складу, де доступ до системи обмежений внутрішньою інфраструктурою:

```env
DJANGO_ENABLE_PASSWORD_VALIDATORS=false
```

Якщо змінна не задана, використовується та сама поведінка (`false`). Для публічного доступу або будь-якого сценарію з доступом з інтернету потрібно увімкнути стандартні валідатори Django:

```env
DJANGO_ENABLE_PASSWORD_VALIDATORS=true
```

Після зміни параметра перезапустіть застосунок, щоб нове налаштування набрало чинності.
