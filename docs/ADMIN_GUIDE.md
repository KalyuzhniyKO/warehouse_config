# Інструкція адміністратора

## 1. Призначення ролі адміністратора
Адміністратор налаштовує користувачів, ролі, довідники, резервне копіювання, принтери та контролює якість даних.

## 2. Як створити адміністратора
Для першого адміністратора виконайте:

```bash
python manage.py createsuperuser
```

## 3. Як створити звичайного користувача
Відкрийте **Керування → Користувачі та ролі** або технічну **Адмін-панель Django** і створіть користувача з логіном та паролем.

## 4. Як призначити групу/роль
Додайте користувача до однієї з груп: **Адміністратор складу**, **Комірник**, **Перегляд / аудитор**.

## 5. Як керувати довідниками
Використовуйте **Керування → Довідники**. Архівуйте непотрібні записи замість фізичного видалення.

## 6. Як архівувати і відновлювати записи
У списку довідника натисніть **Архівувати**. Для повернення відкрийте фільтр **Архівні** і натисніть **Відновити**.

## 7. Як знайти дублікати
```bash
python manage.py find_duplicates
```

## 8. Як ініціалізувати ролі
```bash
python manage.py init_roles
```

## 9. Як оновлювати сервер без втрати даних
Зробіть backup, отримайте новий код, встановіть залежності, виконайте міграції, `compilemessages` за потреби, `collectstatic`, перезапустіть сервіс.

## 10. Як зробити backup
Використовуйте штатний скрипт або дамп MySQL/PostgreSQL. Зберігайте копії окремо від сервера.

## 11. Як відновити backup
Зупиніть сервіс, відновіть дамп у порожню базу, перевірте налаштування підключення, запустіть сервіс і перевірте вхід.

## 12. Як очистити базу для тестового запуску
Працюйте тільки з тестовою базою. Не очищайте бойову базу без підтвердженого backup.

## 13. Як налаштувати принтери CUPS
Встановіть CUPS, додайте принтер у веб-інтерфейсі CUPS, надрукуйте тестову сторінку і перевірте права користувача сервісу.

## 14. Як перевірити логи
Перевіряйте журнал systemd, логи веб-сервера і логи бази даних. Фіксуйте час помилки та користувача.

## 15. Типові проблеми
- **Не працює вхід**: перевірте активність користувача і правильність пароля.
- **Немає прав**: перевірте групу користувача і запустіть `python manage.py init_roles`.
- **Не видно пункт меню**: перевірте, чи користувач увійшов і має потрібну роль.
- **Не працює друк**: перевірте CUPS, принтер, чергу друку і права сервісу.
- **Не працює перемикання мови**: перевірте налаштування `LANGUAGES`, cookies і middleware.
- **Помилка compilemessages**: не комітьте `.mo`; встановіть gettext на сервері і компілюйте там.
- **Помилка MySQL**: перевірте доступи, кодування, мережу, драйвер PyMySQL і залежність cryptography.
