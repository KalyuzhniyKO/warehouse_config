# Запуск складу з нуля

## 1. Підготовка
1. Встановіть залежності проєкту.
2. Налаштуйте `.env` та базу даних.
3. Виконайте міграції.
4. Створіть суперкористувача.
5. Ініціалізуйте ролі.

```bash
python manage.py migrate
python manage.py createsuperuser
python manage.py init_roles
```

## 2. Послідовність заповнення довідників
1. Одиниці виміру.
2. Категорії.
3. Склади.
4. Локації.
5. Отримувачі.
6. Номенклатура.

## 3. Початкові залишки
Після перевірки довідників внесіть фактичні залишки по кожній локації. Не використовуйте архівні довідники для нових операцій.

## 4. Перевірка запуску
Перевірте сторінки залишків, аналітики, довідників і права доступу для кожної ролі.

## Запуск обліку з нуля

1. Створіть одиниці виміру, категорії, склади та локації. Штрихкоди для складів і локацій створяться автоматично.
2. Створіть номенклатуру. Якщо штрихкод не заданий, система створить `ITM`-штрихкод і покаже його у повідомленні після створення.
3. Внесіть стартові кількості через **Початковий залишок** (`/stock/initial/`). Не редагуйте `StockBalance` напряму через форми — залишки змінюються сервісами, які створюють `StockMovement`.
4. Приймайте нові поставки через **Прихід товару** (`/stock/receive/`). Після приходу можна одразу надрукувати етикетку або створити новий прихід.
5. Переміщуйте товар між складами або локаціями через **Складські операції → Переміщення товару** (`/stock/transfer/`).
6. Контролюйте історію у **Рухи товарів** (`/stock/movements/`).
7. Налаштуйте принтери та шаблони етикеток:

```bash
lpstat -p -d
echo test | lp -d PRINTER_NAME
```

8. Друкуйте етикетки зі списку номенклатури, залишків або сторінки результату приходу.

## Переміщення товару

1. Відкрийте **Складські операції → Переміщення товару**.
2. Виберіть товар.
3. Виберіть склад і локацію відправника.
4. Виберіть склад і локацію отримувача.
5. Вкажіть кількість.
6. Натисніть **Перемістити**.
7. Перевірте рух у **Рухи товарів** і залишки у **Залишки**.

## Крок: перша інвентаризація

Після внесення початкових залишків проведіть контрольну інвентаризацію:

1. Відкрийте **Інвентаризація** (`/stock/inventory/`) і створіть нову інвентаризацію по складу або конкретній локації.
2. Внесіть фактичні залишки на сторінці **Підрахунок**.
3. Перевірте різницю на detail-сторінці: система покаже кожний рядок, кількість розбіжностей, суму надлишків і суму нестач.
4. Якщо дані підтверджені, завершіть інвентаризацію. Система автоматично створить `adjustment`-рухи та оновить залишки через складський сервіс.
5. Збережіть результати через **Експорт CSV** або **Експорт XLSX**. CSV має UTF-8 BOM для Excel; XLSX потребує встановленого `openpyxl`.

Аудитор може переглядати й експортувати результати інвентаризації, але не може створювати, редагувати або завершувати її.

## Крок: видача товару зі складу

Після створення номенклатури, складів, локацій і внесення залишків використовуйте **Видача товару** для списання товару з конкретної локації. Вкажіть тип видачі — продаж, ремонтні роботи, виробничі потреби або інше — а також цех / підрозділ і номер документа за потреби.

Система автоматично зменшить залишок, створить рух товарів з типом `out` і заблокує операцію, якщо на локації недостатньо товару.
